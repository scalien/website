What is ScalienDB?

ScalienDB is an enterprise class, open source database for new, high-volume web applications. ScalienDB was designed from the ground up to be a replicated,
scalable, and highly reliable primary database for enterprise data
management use-cases.

ScalienDB is designed for 3-way replication (or more) to ensure high availability in the datacenter. We focus on operational ease-of-use, such as majority failover. This means two of three servers can go down and ScalienDB keeps going without any hiccups or loss in performance; i.e. no need to wake the ops team.

Unlike other NoSQL databases, ScalienDB is built to be a transactional database that utilizes Immediately Consistent replication. This means ScalienDB respects your data and will not ever return stale or inconsistent records. Unlike other NoSQL databases that employ Eventual Consistency, you never have to “repair” or resolve “conflicts” in your ScalienDB database.

As ScalienDB is scalable you can add more servers to support more of your customer’s data, traffic and load. For scalability, ScalienDB divides data into shards and moves these onto new servers as they are added.

The requirements of most enterprise clients leave no room for inferior technology. Designed from the bottom up to for the enterprise ScalienDB offers the following features.

Benefits

Transactions

ScalienDB proves that transactions make just as much sense for a NoSQL database as a traditional SQL database. Transactional properties are codified in the ACID acronym:

Atomicity
either all commands in a transaction are executed or none are.
Consistency
(logical, not to be confused with consistency in replication) - transactions cannot break pre-defined logical consistency rules. This type of consistency does not apply to key-value NoSQL databases.
Isolation
transactions do not see partial writes from other concurrent transactions.
Durability
once committed, transactions stay committed.

Scalability

Scalability is the ability to grow the database as more customers use your web application. It allows you to:

  • Add servers to increase storage capacity
  • Add servers to handle increased web traffic
  • Add servers to handle increased load

Data integrity

Data integrity is the result of Consistency in replication and the use of Transactions. It’s the guarantee that data will be returned in its entirety, as written by the last transaction, without interference from other transactions, as expected by the application programmer.

Majority failover

Majority failover means two out of three servers can fail in a replica set without affecting the availability or performance of ScalienDB. The benefit is reduced operational risk and no late-night emergencies when a server goes offline because of a bad disk, a bad power supply or network equipment failure.

Replication

Replication has become a must-have requirement for databases serving customer-facing web applications. With replication, you have multiple copies of the database stored and maintained, assuring:

  • No data loss in case of disk failure
  • No downtime in case of server failure
  • Decreased reliance on backups

Some applications also require geo-replication. Geo-replication will be present in the next ScalienDB major release.

Consistency

Most other NoSQL databases operate under the premise that high-availability and performance requires Eventual Consistency in replication. Eventual Consistency means that under certain conditions, the database will return stale or inconsistent data.

ScalienDB proves that Eventual Consistency is not required for availability and performance. ScalienDB does not use Eventual Consistency, while maintaining superior failover properties and being at the top of the pack in terms of performance.
ScalienDB achieves this by the pioneering use of the Paxos algorithm for replication.

Performance

ScalienDB has been architected from the beginning to be a high-performance distributed database. It is written in C++ and runs as a native 64-bit application on both Windows and Linux. Additionally, ScalienDB uses an asynchronous, event-based, non-blocking server architecture that assures optimal CPU efficiency and the ability to handle tens of thousands of clients (Windows: IOCP, Linux: Epoll). Unlike other NoSQL databases, ScalienDB can handle a very large number of client connections without eating up your server’s CPU load on either the server or client side.
